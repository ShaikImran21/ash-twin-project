üìã Authentication Flow
1 User Enters Email
User navigates to login page and enters their email address. Form validates email format before proceeding.
2 OTP Generation
System generates a random 6-digit OTP with 5-minute expiry. OTP is logged to console (in production, it would be sent via email).
üîë Generated OTP: 123456 (Check browser console)
3 OTP Verification
User enters the 6-digit code. System validates the OTP and checks expiry time. Supports paste functionality and auto-advance between inputs.
4 Session Creation
On successful verification, a unique session is created with 24-hour validity. Session data is stored in localStorage (use secure cookies in production).
5 Authenticated Access
User is redirected to the main page. Session manager validates authentication state across all pages. User menu appears in navbar showing logged-in status.
